# Markdown Rule — Documentation Standards

## Format
- Use GitHub Flavored Markdown (GFM)
- ATX headings (`#` style), not Setext
- Fenced code blocks with language tag
- Tables with aligned columns

## Structure
- One `#` title per document
- `##` for major sections
- `###` for subsections
- No deep nesting past `####`

## Prose
- English only
- No emojis in documentation or comments
- Active voice, direct tone
- Explain *why*, not *what* (diffs show changes)
